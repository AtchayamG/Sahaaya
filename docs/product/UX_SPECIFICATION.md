# UX Specification

## Information architecture

- Header: brand, provider badge, privacy disclosure.
- Progress rail: Capture → Map → Rehearse → Approve → Verify.
- Household brief: fictional facts and trusted-circle members.
- Continuity map: five responsibility cards with citations and status.
- Rehearsal panel: scenario, gaps, conflicts, and follow-up questions.
- Approval desk: owner/backup proposal, rationale, and decision controls.
- Verified packs: per-person checklist and evidence receipt.

## Visual direction

- Background: warm ivory rather than black agent console.
- Primary ink: near-black blue.
- Accent: restrained turmeric.
- Verified: deep teal.
- Attention: terracotta.
- Cards use subtle borders, generous spacing, and quiet shadows.
- IDs, hashes, and citations wrap without horizontal overflow.

## Interaction rules

- Primary action is singular per stage.
- Rejection and revision are first-class, not hidden destructive actions.
- Loading states explain the current bounded operation.
- No state advances merely because a model responded.
- Export appears only after verification.
- Mobile order matches the workflow; no dense multi-column compression.
