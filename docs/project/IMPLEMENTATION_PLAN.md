# Implementation Plan

## M0 — pre-production

Rules matrix, blueprint, product specification, architecture/contracts, safety, tests, evaluation, submission plan, and clean repository baseline.

## M1 — domain foundation

Next.js scaffold, strict contracts, fictional fixture, deterministic Replay provider, and workflow state machine with focused tests.

## M2 — compiler and provider

Candidate role-pack compiler, deterministic verifier, receipt checksum, API route, and GPT-5.6 provider with fail-closed validation.

## M3 — product experience

Responsive accessible workflow UI, explicit Replay/Live states, conflict and approval interactions, error handling, and export.

## M4 — verification

All unit/route/demo gates, production build, desktop/mobile Playwright/axe, dependency/secret review, visual QA, and bounded Live proof.

## M5 — publication

Public MIT repository, CI, public Replay deployment, and logged-out E2E verification.

## M6 — media

Production screenshots, under-three-minute demo, professional TTS narration, synchronized creator captions, thumbnail, and truthful copy.

## M7 — submission

Apps for Your Life Devpost form, repository/app/video/session links, eligibility details, images, final rules acceptance, submitted badge, and public page audit.
